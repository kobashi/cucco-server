# 上位AI アルゴリズム候補(選定用)

Mock AIの3基本方策(必ずチェンジ / 必ずノーチェンジ / 人数×ランクのマトリクス。
`clients/mock_ai/policies.py`)の次に実装する上位AIの候補。**実装前にこの中から
選定する**。いずれもAIクライアントが受信できる公開情報(捨て札・山札残枚数・
宣言履歴・来歴マップ、`docs/ai-client-guide.md` §3)の範囲内で成立する。

## 必須考慮事項

上位AIが満たすべき判断上の考慮点。

1. **自分が最弱である確率**: 手番時点で、未知カード集合(44枚 − 自分の手札 −
   捨て札)に基づき「オープン時に自分が最弱になる」確率を評価して判断する
2. **チェンジ後の改善確率**: 交換によって最低ランクを回避できる確率(交換相手
   の手札分布)を評価する
3. **山札交換の入手カード制限**: 山札との交換(親の最終手番、または連鎖が末尾
   に達した場合)は、対人交換と入手カードの分布が大きく異なることを考慮する
   - 入手不可: クク(拒否され元の手札維持)・馬・家(引き直し)・人間(引いた
     本人が失格)・猫(引いた者の手札の来歴元が失格。自分の配布札を保持中なら
     自滅)
   - 道化を引いた場合は例外的に**最強**として入手(通常の最弱と正反対)
   - 通常入手できるのは数字札・桶・仮面・獅子のみ。その分布は捨て札・自分の
     手札のカウンティングから正確に計算できる
4. **宣言経緯からの相手手札推定**: チェンジ宣言の並びから各プレイヤーの手札の
   強弱を推定する。特に「チェンジ→(受け手が)ノーチェンジ」の並びは、チェンジ
   した側に**元の弱札よりさらに弱い札**が渡ったことを示唆する(受け手が受領札
   をそのまま保持した = 差し出した札の方が受領札より強かった)
5. **最弱札の伝播と消滅**: 最弱札は連鎖的なチェンジで手番順に山札方向へ流れて
   いき、(a) 山札交換で場から出るか、(b) 途中失格(道化受領など)でディールごと
   消える。この「弱札ウェーブ」の現在位置と消滅済みか否かを追跡し、自分の手番
   位置のリスク評価に反映する
6. **クク宣言タイミングの損得判断**: 低位のMockAIは即時宣言で固定
   (`clients/mock_ai/policies.py`)だが、上位AIは宣言する/しないを自分に有利に
   なるよう判断する。判断材料の例:
   - 交換は強制のため、自分より前の手番の未行動プレイヤーがいる限り、その者の
     カンビオでククを奪われ得る → 奪われる前に宣言する価値がある
   - 逆に1番手(自分を交換対象にできる者がいない位置)では温存し、状況を見て
     宣言/オープン持ち込みを選べる(`docs/rules/play_summary_granpere.md`の定石
     「1番手は温存、2番手以降は即宣言」の一般化)
   - 即時宣言は相手の手札改善機会を奪って現状の弱者を固定する効果もあり、
     「今宣言した場合の敗者分布」と「進行を許した場合の敗者分布」の比較で決める

## 候補アルゴリズム

| 案 | 概要 | 難易度 | 考慮事項の充足 |
|---|---|---|---|
| A. カウンティング確率 | 未知カード集合の一様分布で「最弱確率」と「交換後分布」を計算し閾値判断 | 低 | 1・2・3は可。4・5・6は対象外 |
| B. ベイズ推論 | Aの分布を相手の宣言・拒否イベント・来歴で条件付け更新し、相手ごとの手札事後分布を保持 | 中 | 1〜6すべて(4・5が本体、6は事後分布+手番位置の明示ルール) |
| C. モンテカルロ | 観測と整合する手札配置をサンプリングし、相手をmatrix方策と仮定してディール末尾まで展開、行動別勝率で選択 | 中〜高 | 1・3・5はルール再現で自動的に充足。6は「宣言/パス」も行動候補に含めれば自動。4はBの事後分布を重みに使えば充足 |
| D. チップ期待値 | ディール勝敗でなくゲーム全体のチップ期待値で判断(子供/大人の時間、残チップ、continue判断)。A〜Cの上に載せる層 | 低(拡張) | 判断基準の変更であり1〜6は下層に依存 |
| E. 強化学習 | 評価モード+行動ログ+シード再現を学習環境に自己対戦学習 | 高 | 学習が獲得する(明示実装なし)。ゼミの発展課題向き |

- 案Aは考慮事項4・5・6を満たせないため、**単独では要件を満たさない**(Bの土台
  としては必要)
- 案Bは6条件すべてを明示的なルール/確率更新として実装する。カウンティングの
  教材性が最も高い
- 案Cは山札交換の特殊性(3)・弱札の伝播(5)・クク宣言タイミング(6)をドメイン
  ルールのシミュレーションとして自動的に再現・評価できるのが強み。宣言経緯の
  推定(4)はBの事後分布をサンプリング重みに使う形で取り込む

## 推奨

案B(A内包)を第一候補、精度をさらに上げるなら B+C の組み合わせ。案Dは選定後
どれにでも後付け可能。
