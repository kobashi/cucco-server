# ブラウザクライアント 操作手順書

`clients/web/`の人間向けブラウザクライアントを実際に動かして卓を立て、対局するための
手順書。実装の技術的な詳細は`clients/web/README.md`、実装が満たすべき要件は
`docs/human-client-guide.md`を参照。

## 1. 起動

1. サーバーを起動する(リポジトリルートで):
   ```bash
   source .venv/bin/activate
   python -m cucco.server.app
   ```
   `ws://localhost:8765`で待ち受ける。
2. 別ターミナルで、クライアントを静的ファイルサーバーとして配信する:
   ```bash
   cd clients/web
   python -m http.server 8000
   ```
   (ES Modulesを使っているため`index.html`を直接ダブルクリックしては動かない。
   このサーバーを経由する必要がある)
3. ブラウザで `http://localhost:8000` を開く。デフォルトで
   `ws://<ページを開いたホスト名>:8765`に自動接続する。別ホスト/ポートの
   サーバーに繋ぐ場合は、ブラウザの開発者コンソールで
   `localStorage.setItem("cucco_ws_host", "host:port")`を実行してからページを
   再読み込みする。

## 2. 卓を立てて参加者を募る

1. 名前を入力し、「プレイヤーとして参加」を選んで「つづける」
2. 「卓を作る」→設定フォーム(終了条件・開始チップ枚数・失格カードの開示
   タイミング・馬/家の開示有無)を入力して「作成する」
3. 発行されたプレイルームID(6桁)を参加者に共有する
4. 参加者は「プレイルームIDで参加する」からそのIDを入力して参加
5. 全員が「準備完了」を押すとポットが始まる(最低2人)

一人で動作確認したい場合は、卓を作った後、別ターミナルでモックAIを
参加させると素早く2人以上そろえられる:
```bash
source .venv/bin/activate
python -m clients.mock_ai.mock_ai --name Bot1 --room <プレイルームID> --policy matrix
python -m clients.mock_ai.mock_ai --name Bot2 --room <プレイルームID> --policy always_change
```
AIは参加後すぐに自動で「準備完了」を送るため、人間側が準備完了を押した
時点で(あるいは既にAI側がそろっていれば)ポットが始まる。

複数人でブラウザから確認したい場合は、同じ手順で複数のタブ/ウィンドウを
開き、それぞれ別の名前で参加すればよい。

## 3. 対局中の基本操作

- **どうぞ**: 自分が親のとき、配布直後に表示される。押すと手番が始まる
- **カンビオ/ノンカンビオ**: 自分の手番のとき表示される。カンビオは右隣の
  プレイヤー(親の最終手番のみ山札)との交換を要求する
- **クク宣言/今は宣言しない**: クク札を持っているときだけ、緊急モーダル
  (点滅+カウントダウン)として表示される。他のプレイヤーの画面には何も
  表示されない
- **続行する/離脱する**: 子供の時間(1〜3ディール目)で敗者になったとき、
  必要チップ枚数と現在の残高が並んで表示される

いずれも自分が対象のときしかボタンが出ない(不正操作を未然に防ぐ設計)。

## 4. 確認しておきたいシナリオ

通常の対局フロー(配布→どうぞ→各自の手番→山札交換→オープン→結果)に
加えて、以下は個別に確認しておくとよい。

| シナリオ | 起こし方 | 確認するUI |
|---|---|---|
| 途中失格 | 人間・猫・道化による交換拒否が発生するまで対局する(AI混在で確率を上げるとよい) | 「このディールは失格しました。観戦のみです。」表示、以後そのディールで手番/クク宣言/どうぞの対象から外れること |
| クク宣言 | クク札(44枚中2枚)を引くまで対局する | 点滅する緊急モーダルとカウントダウン、宣言後にディールが即終了すること |
| 続行/離脱 | 子供の時間(1〜3ディール目)で敗者になる | 必要チップ枚数と現在の残高が正しく表示されること |
| 途中の再接続 | 対局中にブラウザタブをリロードする | 「前回の卓の続きがあります」→再接続後、現在の手札・進行状況が復元されること |
| 観戦 | 「観戦者として参加」で同じプレイルームIDから参加する | 手札欄・操作ボタンが一切出ないこと |
| ポット/ゲーム終了 | 誰かのチップが0枚になる、または既定ディール数に達するまで対局する | ポット結果・最終順位画面が表示され、「ロビーに戻る」で最初からやり直せること |
